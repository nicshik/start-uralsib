# Чек-лист проверки аналитики (Demo Mode)

Этот документ поможет убедиться, что все "Три рычага" правильно обвешаны событиями для Яндекс.Метрики.

## 1. Подготовка к Демо
- [ ] Откройте прототип: `npm run dev` (обычно `http://localhost:5173`)
- [ ] Нажмите на иконку **"Зеленая активность" (Metrika Live)** в правом нижнем углу. Это наш "Debug Console".
- [ ] Убедитесь, что список событий пуст или содержит только `page_view: landing`.

## 2. Сценарий проверки "Рычагов"

### Рычаг 1: Вовлечение через Квиз
- [ ] На лендинге нажмите **"Поможем подобрать"**.
- [ ] В консоли должно появиться событие: `quiz_start`.
- [ ] Пройдите квиз до конца (выберите ИП или ООО).
- [ ] Должно появиться событие: `quiz_complete` с параметром `recommendation`.

### Рычаг 2: Магия OCR (Паспорт)
- [ ] Дойдите до Шага 2 (Паспорт).
- [ ] Нажмите **"Сканировать паспорт"**.
- [ ] В консоли: `ocr_started`.
- [ ] Дождитесь завершения имитации (2 сек).
- [ ] В консоли: `ocr_completed` с данными `fieldsAutoFilled: 8`.
- [ ] *Это доказывает, что мы видим успех автоматизации.*

### Рычаг 3: Перенос в Офис
- [ ] Дойдите до последнего экрана (Success).
- [ ] В консоли: `application_submitted`.
- [ ] Нажмите на секретную ссылку **[DEMO: Открыть интерфейс сотрудника]**.
- [ ] В консоли: `page_view: office_agent`.
- [ ] На экране CRM нажмите "Подписать и отправить".
- [ ] В консоли: `manager_workspace_completed`.

## 3. Техническая верификация
- [ ] Все события логируются в `console` с префиксом `[analytics]`.
- [ ] Данные в `AppContext` сохраняются в `localStorage` (проверьте в DevTools -> Application).
- [ ] При перезагрузке страницы на любом шаге должен предлагаться возврат к черновику.
